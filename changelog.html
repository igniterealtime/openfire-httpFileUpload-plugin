  • Updated to HttpFileUploadComponent v1.6.0, which resolves:
    • Issue #41: Add support for (virus)scanning uploaded content

1.3.0 -- (May 23, 2023)

  • Remove unused exclusion to authentication filter

1.2.2 -- (March 5, 2023)

1.2.0 -- (November 30, 2022)

  • Now requires Openfire 4.7.0 or later
  • Updated to HttpFileUploadComponent v1.5.0, which resolves:
    • Issue #38: Update dependencies to latest version
    • Issue #37: fixes #38: Update dependencies to latest version
    • Issue #36: Slot and SecureUniqueId should be Serializable
    • Issue #35: Make slot manager configurable
  • Issue #40: Allow for cluster node specific configuration values.
  • Issue #39: Slot manager should be cluster-aware.
  • Issue #38: Use SystemProperty to replace old style properties.
  • Issue #37: When using non-encrypted scheme, default to non-encrypted port.
  • Issue #36: Add documentation for configuration in a clustered environment.
  • Issue #33: Fix API incompatibilities with upcoming Openfire 4.8.

1.1.5 -- (September 12, 2022)

A meta release to denote compatability with Openfire releases prior to 4.8.0.

1.1.4 -- (November 4, 2021)

  • Updated to HttpFileUploadComponent v1.4.0, which resolves:
    • Updated Guava library to 30.0-jre to address known vulnerabilities with earlier versions.
    • Improve content-type detection of uploaded content.
    • Issue #31: Add new webprotocol option to make the previously hardcoded http protocol configurable.
    • Issue #33: Add new wildcardCORS option that adds CORS headers to relax access restrictions imposed by browsers.
    • Issue #23: Replace UUID with something more secure.
  • Updated to HttpFileUploadComponent v1.3.0, which resolves:
    • Issue #23: Replace UUID with something more secure.

  • Issue #12: Release 1.1.1 fails (unless used on root context).
